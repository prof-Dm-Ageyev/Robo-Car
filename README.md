# 🤖 Проєкт RoboCar

[![Platform](https://img.shields.io/badge/Platform-Arduino%20Nano-blue.svg)](https://www.arduino.cc/en/Main/ArduinoBoardNano)
[![Framework](https://img.shields.io/badge/Framework-PlatformIO-orange.svg)](https://platformio.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Керований через Bluetooth робот-автомобіль на базі Arduino Nano з розширеними можливостями: скриптовані послідовності руху та слідування за лінією, габарити, фари, повороти, стоп-сигнал, задний рух, гудок.

**Автор:** Dmytro Ageyev  
**Поточна версія:** 2.2.1  
**Останнє оновлення:** 20 листопада 2025

---

## 📋 Огляд проєкту

Цей репозиторій містить повний робототехнічний проєкт для побудови та програмування керованого через Bluetooth автомобіля з використанням:

- **Arduino Nano** (ATmega328P @16MHz)
- **Драйвер моторів L298P** (або сумісний H-міст)
- **Bluetooth модуль HC-05/HC-06** (9600 baud)
- **Опціонально:** Інфрачервоні датчики лінії (A0, A4) для автоматичного калібрування

Прошивка надає універсальні опції керування — від базового дистанційного управління до розширених скриптованих послідовностей руху та автоматичного калібрування моторів.

## 🚀 Швидкий старт

### Необхідні компоненти

- **Обладнання:**
  - Arduino Nano (ATmega328P)
  - Драйвер моторів L298P (або сумісний)
  - Bluetooth модуль HC-05 або HC-06
  - 2× DC мотори
  - Джерело живлення (окреме для моторів та логіки рекомендовано)
  - 2× ІЧ датчики лінії

- **Програмне забезпечення:**
  - [VS Code](https://code.visualstudio.com/)
  - [Розширення PlatformIO](https://platformio.org/install/ide?install=vscode)
  - [Arduino IDE](https://www.arduino.cc/en/software/#ide)
  - Bluetooth термінал (Android/iOS) або Serial Monitor

### Встановлення

1. **Clone the repository:**
   ```bash
   git clone https://github.com/prof-Dm-Ageyev/Robo-Car.git
   cd Robo-Car
   ```

2. **Open in VS Code:**
   ```bash
   code RoboCar.code-workspace
   ```

3. **Build and upload firmware:**
   ```powershell
   cd RoboCar
   pio run -t upload
   ```

4. **Test connection:**
   ```powershell
   pio device monitor
   ```

### First Run

1. Power up the robot
2. Pair Bluetooth module (default PIN: `1234` or `0000`)
3. Connect via Bluetooth terminal
4. Send test commands:
   - `5` — set medium speed
   - `F` — move forward
   - `S` — soft stop

## ✨ Ключові можливості

### Базове керування

- **Рух:** Вперед, Назад, Ліворуч, Праворуч, Діагоналі (8 напрямків)
- **Контроль швидкості:** 10 рівнів (0-9) + миттєвий максимум (`q`)
- **Режими зупинки:** Мʼяка зупинка (вибіг) та жорстка зупинка (гальмування)

### Розширені можливості

- **Скриптований рух:** Надсилання послідовностей команд одним рядком
- **Команди затримок:** 11 варіантів таймінгу (25мс до 50с)

## 1. Пін-мапа

| Пін | Макрос / Функція | Призначення |
|-----|------------------|-------------|
| D10 | ML_PWM | ШІМ лівого двигуна |
| D8  | ML_1   | Напрямок лівого двигуна (IN1) |
| D9  | ML_2   | Напрямок лівого двигуна (IN2) |
| D5  | MR_PWM | ШІМ правого двигуна |
| D7  | MR_1   | Напрямок правого двигуна (IN1) |
| D6  | MR_2   | Напрямок правого двигуна (IN2) |
| D2  | BT_RX  | Прийом (RX) мікроконтролера від TX модуля |
| D3  | BT_TX  | Передача (TX) мікроконтролера до RX модуля (потрібен дільник 5V→3.3V) |

> [!NOTE]
> Примітки:
>
> - A6/A7 доступні лише як аналогові входи (без digitalWrite).
> - Для HC-05/HC-06 лінія TX модуля → D2 (5V толерантна), лінія RX модуля ← D3 через дільник (наприклад 1 kΩ + 2 kΩ).

## 2. Огляд функціоналу

Поточна прошивка забезпечує дистанційне керування рухом:

**Основні можливості:**
 Вперед / Назад / Поворот ліворуч / Поворот праворуч / Зупинка (мʼякий `S` або жорсткий `K`)
 Діагональні рухи: вперед-ліворуч (`G`), вперед-праворуч (`H`), назад-ліворуч (`I`), назад-праворуч (`J`)
 10 рівнів швидкості (цифри '0'–'9') + миттєвий максимум ('q')

**Автоматичне калібрування (Line Following):**

> [!IMPORTANT]
> Наявна за умови компіляції з `#define CALIBRATION 1`

 Система PID-контролера для руху вздовж чорної лінії
 2 інфрачервоні датчики лінії (A0 та A4)
 Автоматичне визначення коефіцієнтів компенсації швидкості моторів
 Збереження калібрування в EEPROM
 Окреме калібрування для прямого та зворотного руху

> [!NOTE]
> Детальна інструкція з калібрування: [CALIBRATION_GUIDE.md](./CALIBRATION_GUIDE.md)

## 3. Команди Bluetooth

Параметри порту: 9600 baud, 8N1.

| Команда | Опис |
|---------|------|
| `F` | Рух вперед |
| `B` | Рух назад |
| `L` | Поворот ліворуч (ліве назад, праве вперед) |
| `R` | Поворот праворуч (ліве вперед, праве назад) |
| `G` | Вперед-ліворуч |
| `H` | Вперед-праворуч |
| `I` | Назад-ліворуч |
| `J` | Назад-праворуч |
| `S` | Зупинка (soft) |
| `K` | Зупинка (hard) |
| `0`–`9` | Рівень швидкості (0 ≈ 0%, 9 ≈ ~90% діапазону) |
| `q` | Максимальна швидкість (255) |

> [!NOTE]
> Технічна деталь: формула встановлення швидкості — `map(level, 0, MAX_SPEED_LEVELS, 0, 255)` при `MAX_SPEED_LEVELS = 10`.  
> Це означає, що цифра `9` не дорівнює повному 255 (буде ~229). Для повного максимуму використовуйте `q`.

**Команди затримок (таймінг руху):**

Окремі символи додають паузу (блокуюче `delay(...)`) між командами руху. Завдяки цьому можна надсилати «програму руху» одним рядком через Bluetooth або Serial Monitor.

| Символ | Затримка |
|--------|----------|
| `.` | 25 ms |
| `:` | 50 ms |
| `,` | 100 ms |
| `;` | 250 ms |
| `_` | 500 ms |
| `-` | 1.0 s |
| `=` | 2.5 s |
| `+` | 5.0 s |
| `*` | 10 s |
| `/` | 25 s |
| `%` | 50 s |

> [!TIP]
> Порада: 
> - використовуйте короткі затримки (`. : ,`) у зв’язці з поворотами для плавних траєкторій, а довгі (`- = + * / %`) — для очікування або демонстрації;
> - початку програми поставити `*` або `/` для затримки між завантаженням програми руху та постановкою машинки на трасу.

**Скриптований рух / програмування послідовності:**

Ви можете відправляти команди без розділювачів (або зі пробілами для читабельності — вони будуть проігноровані, якщо не обробляються) у потрібному порядку. Приклад:

```text
*qF--5G_3L,7F=S
```

Розбір:
- `*` — очикування перед початком руху
- `q` — встановити максимальну швидкість (перетвориться у PWM 255)
- `F` — вперед
- `-` `-` — дві паузи по 1 s
- `5` — встановити швидкість "5" (перетвориться у PWM 127)
- `G` — вперед-ліворуч
- `_` — пауза 500 ms
- `3` — швидкість "3" (PWM 76)
- `L` — поворот ліворуч
- `,` — пауза 100 ms
- `7` — швидкість "7" (PWM 179)
- `F` — вперед
- `=` — пауза 2.5 s
- `S` — мʼяка зупинка

Інший приклад довшого сценарію (з очікуванням між фазами):

```text
9F- G- H- S= 6B+ K
```

Це: вперед → пауза 1 c → вперед-ліворуч → пауза 1 c → вперед-праворуч → пауза 1 c → мʼяка зупинка → очікування 2.5 c → назад → пауза 5 c → жорстка зупинка.

Сценарії можна завантажувати як через Bluetooth, так і через Serial Monitor (`pio device monitor`).
